Iluminnare
Case técnico completo
Responsável técnica (consultoria AIOX) — cliente: Luis Felipe, fundador

SaaS de diagnóstico comportamental que precisava evoluir de uma base de código pronta (Lovable) para um produto real, com hospedagem e banco de dados próprios, mantendo o app anterior no ar durante a transição.

  • Migrei a hospedagem completa (Lovable → GitHub próprio + Vercel + Supabase dedicado) sem derrubar o app anterior.
  • Portei o motor de diagnóstico (questionário, pontuação, geração de relatório, PDF e e-mail) de JavaScript puro para TypeScript dentro da aplicação real.
  • Implementei o fluxo completo de produto: onboarding, diagnóstico comportamental, cadastro de projeto, motor de tarefas semanais com liberação sequencial, sistema de pontos e controle de acesso freemium/premium.
  • Conduzi uma auditoria técnica formal (arquitetura, segurança, conformidade, qualidade) sobre o app já em produção com usuários reais, e priorizei os riscos críticos antes de seguir adicionando funcionalidades novas.
  • Corrigi bugs reportados em produção via QA — desde rotas quebradas até lógica de conclusão de tarefas e dados incorretos em painel administrativo.

Case de arquitetura & estratégia
Fundadora & arquiteta do produto

Postos de combustível não têm uma ferramenta que unifique operação física (bombas, PDV, maquininhas) com dados e IA para decisão de gestão. Este case documenta o desenho da arquitetura antes da fase de construção.

  • Defini o modelo de negócio (planos recorrentes por posto) e o roadmap de produto em fases.
  • Conduzi o processo de arquitetura fullstack: Next.js 15 (App Router), Supabase + Drizzle ORM, com estratégia de multi-tenancy via Row Level Security chaveada por organização.
  • Defini padrão Adapter para integrações físicas (bombas, sensores, PDV, maquininhas), evitando acoplamento a um fornecedor específico.
  • Desenhei uma camada de IA transversal e reutilizável entre os módulos do produto, em vez de duplicar lógica por módulo.
  • Modelei cerca de 30 entidades de dados cobrindo a primeira fase do produto.

Rainha da Mídia OS
Case de sistema interno
Fundadora & arquiteta

Como consultora operando sozinha, eu precisava de um painel único para controlar projetos, atendimento e pipeline comercial sem depender de várias ferramentas dispersas — então construí o meu próprio sistema operacional de negócio.

  • Construí a primeira versão (dashboard, CRM/pipeline, central de atendimento com templates, gerador de propostas, financeiro) em HTML/JS para uso imediato.
  • Iniciei a migração para uma segunda versão com stack de produto real (Vite, React, TypeScript, Supabase), incluindo decisão de isolamento de dados por Row Level Security.
  • Defini que o banco de dados dessa v2 seria completamente separado de outros projetos meus, mesmo compartilhando o mesmo padrão de stack — para não misturar dados de negócios diferentes.
